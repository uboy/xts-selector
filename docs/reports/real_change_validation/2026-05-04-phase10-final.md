## Bug fixes (5 total)

### 1. BroadInfraMatch serialization (`5510ac9`)

**Bug**: `cli.py:2213` referenced `e.broad_infra_match.pattern` but `BroadInfraMatch`
dataclass has fields `rule_id`, `rationale`, `fan_out_target`, `false_negative_risk`.
No `pattern` field exists.

**Retrospective effect on Phase 9**: Any PR with broad infra matches would crash
the entire `graph_selection` output with `AttributeError: 'BroadInfraMatch' object
has no attribute 'pattern'`. This means Phase 9 graph resolver silently failed for
a significant fraction of PRs, inflating the apparent "timeout" rate. The 80%
graph batch timeout was partly caused by this crash propagating as a subprocess
error, not by actual performance issues.

**Fix**: Changed `.pattern` to `.rationale`, added `.fan_out_target`.

### 2. Proxy environment variables (`3800323`)

**Bug**: `validate_pr_batch.py` cleared only lowercase `http_proxy`/`https_proxy`
but the environment had all 4 variants set (`HTTP_PROXY`/`HTTPS_PROXY` uppercase).
GitCode API calls went through proxy `http://10.13.105.14:20809`, causing timeouts.

**Retrospective effect on Phase 9**: The 80% graph batch timeout rate was
significantly inflated by proxy-induced API timeouts, not by resolver performance.

**Fix**: Clear all 4 proxy env variants.

### 3. Timing clock mismatch in batch_validate.py

**Bug**: `start_time = time.time()` but `elapsed = time.perf_counter() - start_time`
‚Äî different clocks produced negative elapsed time values.

**Fix**: Changed to `time.perf_counter()` consistently.

### 4. InvertedIndex attribute name

**Bug**: Referenced `inverted.api_names` but correct attribute is `inverted.all_api_names()`
(a method requiring `()`).

**Fix**: Changed to `len(inverted.all_api_names())`.

### 5. INI config parser took wrong section

**Bug**: `_fetch_pr_changed_files` iterated all INI sections and took the first
(`general` ‚Äî has no token), instead of the `gitcode` section.

**Fix**: Replaced manual configparser with `load_ini_git_host_config()` which
correctly resolves the `gitcode` section.

## R-NEW-35 (build graph) ‚Äî DEFERRED

**Why not implemented** (from commit `d42b4af`):

`build.ninja` is not available in the OHOS build output (`out/release/` contains
only `suites/acts/`). XTS `BUILD.gn` files are `group` targets that reference
test HAP packages ‚Äî they do not contain `sources = [...]` lists or C++ source
file references. C++ unittest `BUILD.gn` files do contain source deps, but these
are for gtest targets in `test/unittest/`, not for XTS tests.

**Root cause**: XTS tests are ETS (TypeScript) applications that test through the
public SDK API surface. They are not linked against C++ source files. A build
graph mapping `C++ source ‚Üí test target` cannot bridge to XTS ETS tests.

**Decision**: Build graph approach is not viable for XTS scope. Deferred to
Phase 11 only if senior confirms scope expansion to include C++ unittest coverage.

## R-NEW-37 (internal API indexing) ‚Äî DEFERRED

**Why not implemented** (from commit `d42b4af`):

Survey of internal APIs in ACE engine:
- 1591 `.d.ts` files found, but most are `examples/` boilerplate and `node_modules/`
- 28 meaningful `.ets` internal API definitions in `interfaces/ets/`
- 1925 `.ets` test files in `test/` directory
- Internal APIs already covered by the existing ETS inverted index (2766 APIs)

The existing SDK‚ÜíACE‚Üíconsumer pipeline already indexes these APIs through the ETS
inverted index. A separate "internal API indexer" would duplicate work already done.
